Selenium 3.0 İle Tarayıcıların Çağırılması

Oct 24, 2016

Merhabalar.

Geçtiğimiz günlerde Selenium 3.0 sürümü çıktı ve bir takım değişiklikler ile geldi. Bu değişikliklerden birisi de benim anladığım kadarı ile Selenium geliştirileri tarayıcılar için artık kendileri WebDriver yazmayı bıraktı ve tarayıcıları geliştiren şirketlerin kendilerinden WebDriver geliştirmelerini istedi.

Selenium 3.0 versiyonundan önce örneğin Firefox’u test kodu içerisinden çağırmak için tek yapmamız gereken şu kodu çalıştırmak idi:

WebDriver driver = new FirefoxDriver();
driver.get("http://www.google.com");

Artık Selenium 3.0 ile birlikte Firefox’a ait Driver’ı testi çalıştırmadan önce sistem özellikleri arasına eklememiz gerekiyor. Öncelikle https://github.com/mozilla/geckodriver/releases adresinden Firefox’un çalışmak için kullanacağı GeckoDriver‘ın son sürümünü edinmemiz gerekiyor. İşletim sisteminize özgü sıkıştırılmış dosyayı herhangi bir yere açın. Sıkıştırılmış dosyadan geckodriver.exe çıkacak. Ben C:\Users\ilkay gunel\Downloads dizinine çıkarttım.

Test içerisinde Firefox Driver’ı şu şekilde çağırabiliriz:

import org.junit.Test;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.firefox.FirefoxDriver;

public class ChromeDriverTest {
	@Test
	public void test(){
		System.setProperty("webdriver.gecko.driver","C:\\Users\\ilkay gunel\\Downloads\\geckodriver.exe");
		WebDriver driver = new FirefoxDriver();
		driver.get("http://www.google.com");
	}
}

Eğer tarayıcı olarak Chrome kullanmak istersek de http://chromedriver.storage.googleapis.com/index.html adresinden Chrome Driver’ın son sürümünü edinebiliriz. Oradan da yine bir sıkıştırılmış dosya indirip istediğimiz bir yere açabiliriz. Akabinde kodumuzu da şu şekide güncelleyebiliriz:

import org.junit.Test;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;

public class ChromeDriverTest {
	@Test
	public void test(){
		System.setProperty("webdriver.chrome.driver", "C:\\Users\\ilkay gunel\\Downloads\\chromedriver.exe");
		WebDriver driver = new ChromeDriver();
		driver.get("http://www.google.com");
	}
}

Tabii ki test kodları böyle bir metotdan ibaret olmayacağı için olmayacağı için sistem özelliklerine ekleneme işini daha genel hale getirmek gerekecektir. Bunun için test sınıfında @Before notasyonu ile her test metodu öncesi çalışmasını, @BeforClass notasyonu ile test sınıfı çalışmaya başlamadan önce sistem özelliklerine eklenmesi işini gerçekleştirebiliriz. Ya da bir çeşit tasarım yapıp test metotlarının barındığı sınıfları bu sistem özelliklerine ekleme işlemini yapan bir başka bir sınıfdan kalıtmak sureti ile de bu işi yapabiliriz.

Diğer tarayıcılar için de ilgili kaynaklarından web driver’larının indirilip sistem özelliklerine aynı yolla eklenmesi gerekiyor. Örneğin Opera Tarayıcı için https://github.com/operasoftware/operachromiumdriver/releases adresinden indirme yapıp System.setProperty(“webdriver.opera.driver”, “C:\Users\ilkay gunel\Downloads\operadriver.exe”); şeklinde kodu güncellemek gibi.

Başka bir yazıda görüşene kadar sağlıcakla kalın.

Selam ve Sevgilerimle
